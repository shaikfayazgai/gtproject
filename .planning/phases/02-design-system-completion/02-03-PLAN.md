**DS-38: Evidence Viewer** (`evidence-viewer/`)

This is the most complex governance component. It displays evidence submitted by contributors for review, with contributor identity completely hidden.

```typescript
'use client'
import { useState } from 'react'
import { Tabs, TabsList, TabsTrigger, TabsContent } from '../tabs/tabs'
import { Highlight, themes } from 'prism-react-renderer'
import { Code, FileText, Link as LinkIcon, Video, Type, Download, ExternalLink } from 'lucide-react'
import { cn } from '../../lib/utils'
```

Types:
```typescript
type EvidenceType = 'code' | 'document' | 'link' | 'video' | 'text'

interface CodeEvidence {
  type: 'code'
  language: string
  code: string
  filename?: string
}

interface DocumentEvidence {
  type: 'document'
  filename: string
  fileSize: string    // e.g., "2.4 MB"
  fileType: string    // e.g., "PDF", "DOCX"
  downloadUrl: string
}

interface LinkEvidence {
  type: 'link'
  url: string
  title?: string
  description?: string
}

interface VideoEvidence {
  type: 'video'
  url: string         // embed URL (YouTube, Vimeo)
  title?: string
}

interface TextEvidence {
  type: 'text'
  content: string
}

type Evidence = CodeEvidence | DocumentEvidence | LinkEvidence | VideoEvidence | TextEvidence

interface EvidenceViewerProps {
  evidence: Evidence[]
  className?: string
}
```

Implementation:
- Uses Tabs from 02-01 for the 5-tab layout
- Each tab has an icon (Code, FileText, LinkIcon, Video, Type from lucide)
- Only tabs with matching evidence are shown (filter `evidence` array by type)
- **Code tab:** Use `Highlight` from prism-react-renderer with `themes.github` theme. Render with line numbers. Wrap in `bg-bg-dashboard rounded-inner overflow-x-auto` container. If `filename` is provided, show it as a header bar.
- **Document tab:** Show file icon, filename, file size, file type. Download link with Download icon. Use a card-style container. Do NOT embed PDF viewer -- just show metadata and download link.
- **Link tab:** Show URL as a card with external link icon. If title/description provided, show them. Link opens in new tab.
- **Video tab:** Render iframe with `w-full aspect-video rounded-inner` classes. Use the embed URL directly.
- **Text tab:** Render content in a `prose` container with `text-sm font-body text-text-body whitespace-pre-wrap`.

CRITICAL: There is NO contributor name, avatar, initials, ID, or any identifying information ANYWHERE in this component. The Evidence type does not even have a contributor field. This is by design -- evidence is reviewed blind.

Story:
- Story with all 5 evidence types (code in TypeScript, a document, a link, a video, and text)
- Story with only code evidence
- Story showing the blind review constraint (no identity visible)
